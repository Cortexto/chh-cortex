# CHH Cortex — Operating contract

## Raw vs wiki (read this once)

| Path | Role |
|------|------|
| **`raw/`** | **Source inbox** — emails, meetings, RFP notes, drafts. Land first; append-only. |
| **`wiki/`** | **Curated truth** — people, rules, lessons. Promote only with curator approval + `Source(s)`. |

Full explanation and pain-point map: **`CHH-ROADMAP.md`** (root — start here for Paul/William).

## Other folders

| Path | Purpose |
|------|---------|
| `skills/` | Runnable workflows (grant, triage, email, review) |
| `_system/` | Install guide only (`CHH-INSTALL.md`) |
| `TASKS.md` | Shared CHH task list |

## Promotion rules

1. Land evidence in `raw/` before wiki promotion
2. Every `wiki/` fact needs `Source(s)`
3. When facts change, **update or supersede** wiki — do not leave stale “current” claims

## Pilot guardrails

- No PII/PHI in shared vault
- Red-team before paste-ready grant text
- Mark uncertain claims `verify` or `needs source`

See `wiki/partnerships/chh.md` and `wiki/concepts/workflow/chh-workflows-guide.md`.
